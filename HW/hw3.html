Use Gecode to support a basic scheduler for instructors and class sections at a set of rooms. The Grading section spells out the details of what you need to handle.

Some simplifications, clarifications:

1.      You can assume all time slots are 2 hours long. It is fine to have non-overlapping time slots.

2.      You can assume teachers have to teach the same number of sections

3.      You can assume all rooms are the same size, and therefore all sections can be assigned to any room

4.      Your code is only creating a schedule for one term there is no registration

Useful Gecode features:

         The support for limiting integer variables to subsets of all integers. The subsets can be ranges of values, like 1 to 10, or sets of values, like {1, 4, 9}. For example, see : http://www.gecode.org/doc-latest/reference/classGecode_1_1IntVarArray.html (the last constructor) and 4.4.1 of the manual (http://www.gecode.org/doc-latest/MPG.pdf)

         There are methods to extract the values of Gecode variables for display purpose. For example, IntVar has a val() method that returns the value as an int.

 

 

Grading:

Correctness: (weights are for 4810)

         10% to get 1 instructor into different time slots, 1 room, same course for at least 2 sections each

         10% to produce meaningful output (ex: 4pm N Sci 112 CS 4590 Brown)

         10% to handle multiple instructors (but each instructor still has a limit on how many sections they can be assigned)

         10% to handle multiple rooms (so there will at least sometimes be more than one section offered in the same time slot)

         10% to handle multiple courses

         10% to handle multiple departments instructors can only be assigned courses from one department

         10% to handle requirement that while there may be more than one section of the same course, there must be at least one section of each course

         10% to handle instructors only being given sections from a subset of their departments courses (these are the ones they prefer to teach). This constraint should be handled as a soft constraint, and you should demonstrate that for data where you will have to violate this constraint at least twice, you violate a minimum of these. Its ok if its one instructor who has to teach multiple courses not on his/her list of preferred courses.

 

6810 weight adjustments, additional requirements:
